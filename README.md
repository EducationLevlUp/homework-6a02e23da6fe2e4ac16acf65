# Агент с RAG-памятью

AI-агент с локальным RAG-хранилищем знаний на базе **ChromaDB** и **Ollama**.
Агент умеет искать и сохранять информацию в векторной базе через инструменты
`search_knowledge_base` и `add_to_knowledge_base`.

---

## Архитектура

```
┌──────────────┐       ┌──────────────────┐       ┌──────────────┐
│   CLI (main) │──────▶│  LangChain Agent  │──────▶│  ChromaDB    │
│  /add        │       │  (create_agent)   │       │  (local)     │
│  /search     │◀──────│  + RAG tools      │◀──────│              │
│  /quit       │       │                   │       └──────────────┘
└──────────────┘       └────────┬─────────┘
                                │
                    ┌───────────▼──────────┐
                    │  Ollama Embeddings   │
                    │  (nomic-embed-text)  │
                    └──────────────────────┘
```

## Структура проекта

| Файл | Назначение |
|------|-----------|
| `main.py` | Интерактивный CLI-клиент с агентом |
| `vector_store.py` | ChromaDB-клиент, коллекция, векторное хранилище |
| `tools.py` | `@tool` инструменты: `search_knowledge_base`, `add_to_knowledge_base` |
| `chunking.py` | `RecursiveCharacterTextSplitter` с метаданными |
| `init_db.py` | Скрипт загрузки документов из директории |
| `requirements.txt` | Python-зависимости |

## Установка

### 1. Внешние сервисы

```bash
# Ollama: загрузить модели
ollama pull llama3
ollama pull nomic-embed-text
```

> **Примечание:** В актуальных версиях Ollama тег `llama3` может
> перенаправляться на `llama3.1` или `llama3.2`. Это нормально —
> код будет работать с любой доступной версией.

### 2. Python-зависимости

```bash
pip install -r requirements.txt
```

## Быстрый старт

### 1. Инициализация базы знаний

Создайте директорию с текстовыми файлами и запустите скрипт:

```bash
# Пример: создать тестовые документы
mkdir -p sample_docs
echo "Python — это высокоуровневый язык программирования." > sample_docs/python.txt
echo "ChromaDB — векторная база данных для поиска по эмбеддингам." > sample_docs/chromadb.txt

# Загрузить документы в базу
python init_db.py sample_docs
```

### 2. Запуск интерактивного клиента

```bash
python main.py
```

Доступные команды:

| Команда | Описание |
|---------|----------|
| `/add <текст> \| <заголовок>` | Добавить документ в базу знаний |
| `/search <запрос>` | Семантический поиск по базе знаний |
| `/quit` | Выход из программы |
| `<любой текст>` | Запрос агенту (агент сам решит, использовать ли RAG) |

### Пример сессии

```
>>> /add "LangChain — фреймворк для построения AI-приложений." | LangChain
```

## Технические детали

### Векторное хранилище

- **ChromaDB** работает в локальном режиме (данные хранятся в папке `chroma_data/`)
- Дистанция по умолчанию: `cosine` (оптимальна для `nomic-embed-text`)
- Размерность эмбеддингов: 768 (определяется моделью `nomic-embed-text`)

### Чанкинг

- `RecursiveCharacterTextSplitter` с `chunk_size=1000`, `chunk_overlap=200`
- Метаданные каждого чанка: `title`, `source` (путь к файлу)

### Агент

- Создан через `create_react_agent` (современный API LangChain v0.2+)
- Системный промпт инструктирует агента использовать RAG-инструменты
- LLM: `llama3` через Ollama

## Настройка

Переменные окружения (опционально):

| Переменная | По умолчанию | Описание |
|-----------|-------------|----------|
| `CHROMA_DATA_DIR` | `chroma_data` | Путь к данным ChromaDB |
| `CHROMA_COLLECTION` | `rag_collection` | Имя коллекции |
| `EMBEDDING_MODEL` | `nomic-embed-text` | Модель эмбеддингов |

---

## Безопасность

- Все данные хранятся локально (ChromaDB local mode)
- Никакие запросы не отправляются во внешние сервисы
- Токены и секреты не используются
